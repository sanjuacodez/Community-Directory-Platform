# Database Schema (Supabase)

Same schema as previously documented. Supabase uses standard PostgreSQL.

## Schema Compatibility

The Prisma schema in `backend/prisma/schema.prisma` is the canonical definition. For Supabase migration, it converts to:

1. **Enums** → PostgreSQL ENUM types (generated by Supabase migration)
2. **UUID** → `uuid` type with `gen_random_uuid()` default
3. **Timestamps** → `timestamptz` with `now()` defaults
4. **Relations** → Foreign key constraints with CASCADE deletes
5. **Uniques** → Unique constraints (slug, email, etc.)

## Supabase-Specific Additions

### Row-Level Security
All tables get RLS policies for role-based access.

### Auth Schema
Supabase adds `auth.users` table automatically for authentication. Our `users` table is replaced by Supabase Auth. Existing `roles` and `user_roles` tables remain for RBAC.

### Real-Time (Future)
Tables can optionally enable Supabase Realtime for live updates in the UI.
